## FP-016 — Condivisione tramite link

---

### 1. Obiettivo funzionale

La funzionalità “Condivisione tramite link” consente all’utente di generare un link condivisibile contenente uno scenario di stima Microsoft Copilot Cowork Credits.

L’obiettivo è permettere la condivisione rapida di una simulazione senza backend, senza database e senza account utente.

Il link deve contenere i dati necessari per aprire lo scenario in modalità preview, ad esempio:

* company minima;
* scenario;
* segmenti;
* usage profile custom usati;
* task preset custom usati;
* modelli custom usati;
* assumption pack custom o snapshot;
* funding plan;
* calculation result;
* warning;
* metadata;
* disclaimer.

La funzionalità deve supportare casi d’uso come:

* inviare una simulazione a un collega;
* condividere una bozza con un cliente;
* far revisionare uno scenario;
* passare una stima tra browser;
* allegare una simulazione a un messaggio;
* aprire uno scenario senza importare un file JSON.

Il link non deve salvare automaticamente i dati nel browser di chi lo apre. L’utente destinatario deve prima vedere una preview e poi decidere se salvare localmente.

---

### 2. Attori coinvolti

Gli attori principali sono:

* Consulente;
* Solution architect;
* Sales specialist;
* Account manager;
* Responsabile Finance;
* Responsabile IT;
* Cliente finale;
* Utente che crea una simulazione;
* Utente che riceve una simulazione;
* Utente che vuole condividere una bozza;
* Utente che lavora senza backend o login.

---

### 3. Prerequisiti

Per usare la condivisione tramite link devono essere disponibili:

* scenario salvato o draft condivisibile;
* dati minimi dello scenario;
* export service;
* import/preview service;
* compressione payload;
* encoding URL-safe;
* validation service;
* schema version;
* dependency collector;
* storage locale;
* gestione limiti dimensione URL;
* fallback copia manuale.

La funzionalità deve funzionare in un’app statica, anche se pubblicata su GitHub Pages.

---

### 4. Trigger

La funzionalità viene attivata quando:

* l’utente clicca “Share scenario”;
* l’utente clicca “Copy scenario link”;
* l’utente clicca “Share draft”;
* l’utente apre un link contenente payload scenario;
* l’app rileva un payload nel fragment URL;
* l’utente decide di salvare localmente uno scenario condiviso;
* il payload del link è corrotto o troppo grande;
* il browser blocca la Clipboard API;
* l’utente prova a condividere dati con assumption custom;
* l’utente prova a condividere scenario stale o invalid.

---

### 5. Descrizione funzionale dettagliata

Il sistema deve generare un link che contiene un payload serializzato, compresso e codificato in formato URL-safe.

Il payload deve essere inserito preferibilmente nel fragment della URL.

Esempio concettuale:

```text id="b2ibcq"
https://example.com/#/shared?payload=<compressedPayload>
```

Oppure:

```text id="9xfwfs"
https://example.com/#share=<compressedPayload>
```

L’uso del fragment è preferibile perché, in condizioni normali, il fragment non viene inviato al server durante la richiesta HTTP.

Quando un utente apre il link, l’app deve:

1. rilevare il payload;
2. decodificarlo;
3. decomprimerlo;
4. validarlo;
5. mostrare una preview;
6. indicare warning e dati inclusi;
7. permettere di:

   * aprire in sola lettura;
   * salvare localmente;
   * duplicare;
   * esportare JSON;
   * annullare.

Lo scenario condiviso non deve essere salvato automaticamente.

### 8. Regole di business

#### BR-FP016-001 — Condivisione senza backend

La generazione e apertura del link devono funzionare senza backend.

Il sistema non deve salvare payload su server.

---

#### BR-FP016-002 — Payload nel fragment URL

Il payload deve essere inserito preferibilmente nel fragment della URL.

Questo riduce il rischio che il payload venga inviato al server durante la navigazione.

---

#### BR-FP016-003 — Link non è backup durevole

La UI deve chiarire che il link serve per condivisione rapida.

Per backup o archiviazione completa, l’utente deve usare Export JSON.

---

#### BR-FP016-004 — Scenario condiviso non salvato automaticamente

Quando l’utente apre un link condiviso, lo scenario deve essere mostrato in preview.

Non deve essere salvato localmente finché l’utente non clicca “Save locally”.

---

#### BR-FP016-005 — Salvataggio locale esplicito

Se l’utente decide di salvare lo scenario condiviso, il sistema deve:

* generare nuovi ID;
* rimappare riferimenti;
* marcare lo scenario come imported/shared;
* salvare in localStorage;
* mostrare conferma.

---

#### BR-FP016-006 — Dati minimi inclusi

Il link deve includere dati sufficienti a visualizzare lo scenario anche su un browser senza dati locali.

Per uno scenario devono essere inclusi almeno:

* company minima;
* scenario;
* segmenti;
* funding plan;
* assumption snapshot o pack;
* calculation result, se disponibile.

---

#### BR-FP016-007 — Dipendenze custom incluse

Se lo scenario usa oggetti custom, il payload deve includerli.

Esempi:

* custom assumption pack;
* custom usage profile;
* custom task preset;
* custom model.

---

#### BR-FP016-008 — System data non necessariamente inclusi

I dati system standard possono non essere inclusi integralmente se ricostruibili dall’app.

Tuttavia il payload deve contenere riferimenti o snapshot sufficienti per evitare ambiguità.

---

#### BR-FP016-009 — Schema version obbligatoria

Ogni payload condiviso deve contenere schema version e payload version.

---

#### BR-FP016-010 — Validazione all’apertura

Il payload deve essere validato prima di mostrare la preview.

Se il payload non è valido, il sistema deve mostrare errore e non salvare nulla.

---

#### BR-FP016-011 — Gestione schema vecchio

Se il link contiene payload con schema precedente, il sistema deve tentare migrazione.

Se la migrazione riesce, mostra warning.

Se fallisce, blocca apertura.

---

#### BR-FP016-012 — Gestione schema futuro

Se il link contiene payload creato da una versione futura non supportata, il sistema deve bloccare apertura o mostrare warning forte.

Per MVP: bloccare apertura se incompatibile.

---

#### BR-FP016-013 — Compressione payload

Il payload deve essere compresso quando possibile.

L’obiettivo è ridurre la lunghezza del link.

---

#### BR-FP016-014 — Encoding URL-safe

Il payload deve essere codificato in formato URL-safe.

Esempio:

* base64url;
* rimozione caratteri non sicuri;
* niente spazi;
* niente newline.

---

#### BR-FP016-015 — Limite dimensione link

Il sistema deve controllare la lunghezza del link.

Se supera una soglia configurata, deve mostrare warning.

Soglie consigliate:

* soft limit: 2.000 caratteri;
* warning limit: 4.000 caratteri;
* hard limit: 8.000 caratteri.

Se il link è troppo grande, il sistema deve proporre Export JSON.

---

#### BR-FP016-016 — Clipboard API con fallback

Il sistema deve provare a copiare il link negli appunti.

Se la Clipboard API non è disponibile o viene bloccata, deve mostrare il link in un campo selezionabile manualmente.

---

#### BR-FP016-017 — Privacy warning

Prima di generare link, il sistema deve avvisare che il link può contenere dati inseriti dall’utente.

Esempi:

* company name;
* scenario name;
* note;
* assunzioni custom;
* stime economiche.

---

#### BR-FP016-018 — Link non criptato

Il link non deve essere presentato come sicuro o criptato, salvo implementazione reale di cifratura.

Il sistema deve indicare:

“Chiunque abbia il link può leggere i dati contenuti.”

---

#### BR-FP016-019 — Opzione exclude notes

L’utente dovrebbe poter escludere note e descrizioni libere dal payload.

Per MVP può essere un’opzione semplice:

* include notes: sì/no.

---

#### BR-FP016-020 — Dati demo

Se lo scenario condiviso contiene dati demo, il payload deve marcarli come demo.

---

#### BR-FP016-021 — Scenario stale

Se lo scenario ha calculation result stale, il link può essere generato, ma deve includere lo stato stale.

Il destinatario deve vedere warning.

---

#### BR-FP016-022 — Scenario invalid

Se lo scenario è invalid, il sistema può consentire condivisione come draft, ma non come scenario reviewed.

Deve mostrare warning.

---

#### BR-FP016-023 — Calculation result incluso

Se disponibile, il calculation result deve essere incluso per permettere preview immediata.

Il destinatario può ricalcolare dopo salvataggio locale.

---

#### BR-FP016-024 — Assumption snapshot incluso

Il payload deve includere assumption snapshot o pack sufficiente per spiegare come il risultato è stato calcolato.

---

#### BR-FP016-025 — Nessuna scadenza reale senza backend

Il sistema non deve promettere link con scadenza reale se non esiste backend.

Può includere `createdAt` e mostrare warning se il link è vecchio, ma non può invalidarlo server-side.

---

#### BR-FP016-026 — Link generato da dati locali

Il link è generato dai dati locali correnti.

Se i dati cambiano dopo la generazione, il link precedente non si aggiorna.

---

#### BR-FP016-027 — Rimozione payload dalla barra URL

Dopo apertura e preview, il sistema può offrire di rimuovere il payload dalla barra URL per privacy locale.

Esempio:

* “Clean URL”.

---

#### BR-FP016-028 — Export JSON come fallback

Se il payload è troppo grande o la generazione link fallisce, il sistema deve proporre export JSON.

---

#### BR-FP016-029 — Import da link come shared

Gli oggetti creati da link devono avere source:

```text id="9bwfpz"
source = "shared"
```

o:

```text id="xz7lt8"
source = "imported"
```

---

#### BR-FP016-030 — Non modificare dati locali in preview

L’apertura del link in preview non deve modificare storage locale, preferenze o scenari esistenti.

---

### 11. Flusso principale — Generazione link scenario

1. L’utente apre uno scenario.
2. Clicca “Share”.
3. Il sistema mostra modal con opzioni.
4. L’utente sceglie:

   * include calculation result;
   * include notes;
   * include custom assumptions;
   * include warning.
5. Il sistema mostra privacy notice.
6. L’utente conferma.
7. Il sistema costruisce payload.
8. Il sistema comprime e codifica.
9. Il sistema verifica lunghezza URL.
10. Se valida, copia il link negli appunti.
11. Mostra conferma.
12. Se copia fallisce, mostra campo con link manuale.

---

### 12. Flusso principale — Apertura link condiviso

1. L’utente apre un link con payload.
2. L’app rileva `share`.
3. Decodifica e valida payload.
4. Mostra preview scenario.
5. Mostra:

   * company;
   * scenario;
   * costi;
   * crediti;
   * assumption pack;
   * warning;
   * data creazione link.
6. L’utente può:

   * Save locally;
   * Export JSON;
   * Open read-only;
   * Cancel.
7. Se sceglie Save locally, il sistema importa come nuovo.
8. Il sistema salva in localStorage.
9. La UI mostra scenario importato.

---

### 13. Flussi alternativi

#### A1 — Link troppo grande

1. L’utente clicca Share.
2. Il sistema genera payload.
3. La URL supera hard limit.
4. Il sistema blocca link.
5. Propone Export JSON.

---

#### A2 — Warning dimensione link

1. La URL supera warning limit ma non hard limit.
2. Il sistema mostra warning.
3. L’utente può copiare comunque o usare JSON.
4. Il link viene generato se confermato.

---

#### A3 — Clipboard bloccata

1. Il sistema prova a copiare il link.
2. Il browser blocca clipboard.
3. Il sistema mostra link in textarea.
4. L’utente copia manualmente.

---

#### A4 — Payload corrotto

1. L’utente apre link.
2. Decodifica o parsing fallisce.
3. Il sistema mostra errore.
4. Nessun dato viene salvato.

---

#### A5 — Schema vecchio

1. Payload usa schema precedente.
2. Il sistema tenta migrazione.
3. Se riesce, mostra warning.
4. Mostra preview.

---

#### A6 — Schema futuro

1. Payload usa schema non supportato più recente.
2. Il sistema blocca apertura.
3. Mostra messaggio.

---

#### A7 — Scenario stale condiviso

1. Payload contiene result stale.
2. La preview mostra warning.
3. L’utente può salvare e ricalcolare.

---

#### A8 — Scenario invalid condiviso

1. Payload contiene scenario invalid.
2. Il sistema lo mostra come draft.
3. Non permette salvataggio come reviewed.
4. Mostra errori.

---

#### A9 — Notes escluse

1. L’utente genera link con include notes = false.
2. Il sistema rimuove descrizioni e note libere dal payload.
3. Il link viene generato con dati ridotti.

---

#### A10 — Dati custom mancanti

1. Payload referenzia un custom object non incluso.
2. Il sistema mostra warning.
3. Può usare fallback o bloccare preview.
4. Per MVP: bloccare se dipendenza obbligatoria.

---

#### A11 — Link aperto su browser con dati locali esistenti

1. L’utente apre link su browser con dati già presenti.
2. Il sistema mostra preview.
3. Non modifica dati esistenti.
4. Se l’utente salva, importa come nuovo.

---

#### A12 — Utente annulla preview

1. L’utente apre link.
2. Vede preview.
3. Clicca Cancel.
4. Nessun dato viene salvato.
5. Il sistema può pulire URL.

### 14. Validazioni

#### Validazioni bloccanti

* scenario mancante;
* payload non serializzabile;
* payload non comprimibile;
* URL oltre hard limit;
* shareType mancante;
* schemaVersion mancante;
* payloadVersion mancante;
* encoded payload non valido;
* decode fallito;
* decompress fallito;
* JSON parse fallito;
* data object mancante;
* scenario condiviso senza ID;
* scenario condiviso senza company minima;
* riferimenti obbligatori non risolti;
* assumption snapshot mancante;
* schema futuro incompatibile;
* checksum non valido, se previsto.

---

#### Warning non bloccanti

* URL oltre soft limit;
* URL oltre warning limit;
* scenario stale;
* scenario draft;
* scenario invalid condiviso come draft;
* assumption custom incluse;
* note incluse;
* dati demo;
* link vecchio;
* payload senza calculation result;
* funding mancante;
* pack deprecated;

### 20. Criteri di accettazione

#### AC-001 — Generazione link scenario

Given uno scenario condivisibile,
when l’utente clicca Share e conferma,
then il sistema genera un link contenente payload scenario.

---

#### AC-002 — Payload nel fragment

Given viene generato un link,
when l’utente lo visualizza,
then il payload è inserito nel fragment URL o in area equivalente client-side.

---

#### AC-003 — Link copiato

Given il browser consente clipboard,
when il link viene generato,
then il sistema lo copia negli appunti.

---

#### AC-004 — Clipboard fallback

Given il browser blocca clipboard,
when il link viene generato,
then il sistema mostra il link in un campo copiabile manualmente.

---

#### AC-005 — Privacy notice

Given l’utente genera un link,
when il sistema mostra opzioni share,
then visualizza un avviso che chiunque abbia il link può vedere i dati inclusi.

---

#### AC-006 — Link troppo grande

Given il payload supera il limite massimo URL,
when l’utente prova a generare link,
then il sistema blocca generazione e propone Export JSON.

---

#### AC-007 — Warning link lungo

Given il link supera la soglia warning ma non hard limit,
when viene generato,
then il sistema mostra warning e permette conferma.

---

#### AC-008 — Apertura link valido

Given l’utente apre un link condiviso valido,
when l’app carica,
then il sistema mostra preview dello scenario.

---

#### AC-009 — Nessun autosalvataggio

Given l’utente apre un link condiviso,
when visualizza la preview,
then lo scenario non viene salvato localmente automaticamente.

---

#### AC-010 — Save locally

Given l’utente visualizza preview condivisa,
when clicca Save locally,
then il sistema importa lo scenario con nuovi ID e lo salva localmente.

---

#### AC-011 — Import come shared

Given uno scenario viene salvato da link,
when il salvataggio termina,
then lo scenario è marcato come shared/imported.

---

#### AC-012 — Remap riferimenti

Given lo scenario condiviso viene salvato localmente,
when vengono generati nuovi ID,
then segmenti, funding, result e riferimenti interni restano coerenti.

---

#### AC-013 — Payload corrotto

Given l’utente apre link con payload corrotto,
when il sistema prova a decodificarlo,
then mostra errore e non salva dati.

---

#### AC-014 — Schema vecchio

Given il link contiene schema precedente supportato,
when viene aperto,
then il sistema migra il payload e mostra warning.

---

#### AC-015 — Schema futuro

Given il link contiene schema non supportato più recente,
when viene aperto,
then il sistema blocca apertura.

---

#### AC-016 — Notes escluse

Given l’utente disattiva include notes,
when genera link,
then note e descrizioni libere non sono incluse nel payload.

---

#### AC-017 — Scenario stale condiviso

Given lo scenario ha calculation result stale,
when viene condiviso,
then il link può essere generato ma la preview mostra warning stale.

---

#### AC-018 — Scenario invalid condiviso

Given uno scenario è invalid,
when l’utente lo condivide,
then il sistema lo condivide solo come draft o mostra warning bloccante secondo configurazione.

---

#### AC-019 — Export JSON fallback

Given la condivisione link non è possibile,
when il sistema mostra errore,
then propone Export JSON come alternativa.

---

#### AC-020 — Clean URL

Given l’utente apre una preview condivisa,
when clicca Clean URL,
then il payload viene rimosso dalla barra indirizzi senza salvare dati.

---

#### AC-021 — Dati locali non modificati in preview

Given l’utente apre un link condiviso,
when non clicca Save locally,
then lo storage locale resta invariato.

---

#### AC-022 — Link con custom assumptions

Given uno scenario usa custom assumption pack,
when viene condiviso,
then il payload include pack o snapshot necessari e la preview mostra badge custom.

---

#### AC-023 — Link senza calculation result

Given uno scenario non ha calculation result,
when viene condiviso come draft,
then la preview mostra dati input e indica che il risultato deve essere calcolato.

---

#### AC-024 — Shared scenario export JSON

Given l’utente apre una preview da link,
when clicca Export JSON,
then il sistema esporta il payload in file JSON senza salvarlo localmente.

---

#### AC-025 — Funzionamento senza backend

Given l’app è pubblicata come sito statico,
when l’utente genera o apre un link condiviso,
then tutto avviene nel browser senza chiamate server.

### 21. Priorità

Priorità funzionale: **Should have per MVP**, **Must have per esperienza collaborativa completa**.

La condivisione tramite link non è strettamente necessaria per il calcolo, ma aumenta molto il valore pratico dell’app.

Permette di:

* condividere una simulazione rapidamente;
* evitare file allegati in casi semplici;
* far revisionare scenari;
* supportare collaborazione leggera;
* aprire scenari in altri browser;
* mantenere l’app serverless.

Tuttavia, per dati grandi o backup, il JSON export resta più affidabile.

---

### 22. Note UX

La UX deve essere estremamente chiara sui limiti della condivisione link.

Principi UX:

* pulsante Share visibile nello scenario report;
* privacy warning prima della copia;
* opzione exclude notes;
* indicatore dimensione link;
* suggerimento JSON se link lungo;
* preview obbligatoria per il destinatario;
* nessun autosave;
* CTA Save locally chiara;
* badge Shared scenario;
* possibilità Clean URL;
* messaggi chiari se link è corrotto o troncato;
* evitare promesse di sicurezza o scadenza se non implementate.

### 24. Definizione di completato

Il FP-016 può essere considerato completo quando:

* l’utente può generare link da scenario;
* il payload viene serializzato;
* il payload viene compresso;
* il payload viene codificato in modo URL-safe;
* il payload viene inserito nel fragment URL;
* il sistema controlla la lunghezza del link;
* link troppo grandi vengono bloccati;
* il sistema propone JSON export come fallback;
* la Clipboard API è supportata;
* esiste fallback copia manuale;
* la UI mostra privacy warning;
* l’utente può escludere note dal link;
* l’app rileva link condiviso all’avvio;
* l’app decodifica e valida payload;
* l’app mostra preview prima del salvataggio;
* lo scenario condiviso non viene salvato automaticamente;
* l’utente può salvare localmente con nuovi ID;
* i riferimenti interni vengono rimappati;
* lo scenario importato è marcato shared/imported;
* link corrotti vengono gestiti senza crash;
* schema vecchio viene migrato se supportato;
* schema futuro incompatibile viene bloccato;
* l’utente può pulire la URL;
* l’utente può esportare JSON dalla preview;
* tutto funziona senza backend e resta compatibile con GitHub Pages.
